What it does

A focused, production-ready library designed for one task: ranking glass–film interface candidates before expensive DFT simulations.

The Problem

Depositing a thin film on glass requires choosing the right material and deposition conditions. DFT calculations for each candidate can take days per system. You need a way to triage candidates first.

The Solution

surface_dipole_library computes the dipole moment of each surface atom group, then evaluates the dipole–dipole interaction energy for every substrate–film atom pair using classical electrostatics — all in under a second.

The Outcome

A ranked DataFrame sorted from the strongest attractive pair to the strongest repulsive pair, plus bulk dipole moments, best interaction summary, and statistics. Use this to rank candidates and guide DFT input — only simulate the most promising interfaces.

How it works

Five deterministic steps from atom input to ranked interaction table.

1
Input Atoms

Define film atoms with element symbol, partial charge (e), and Cartesian position (Å). The glass substrate is fixed automatically.

2
Dipole Moments

Compute μ = Σqiri for both surfaces. Quantifies the net charge displacement of each group.

3
Local Dipoles

Per-atom effective dipole relative to group centroid: μi = qi(ri − rc). Origin-independent by design.

4
Pairwise Energies

All N×M pairs evaluated with the dipole–dipole formula. Distance cutoff filters out long-range negligible interactions.

5
Ranked Table

DataFrame sorted by energy (most negative = rank 1). Green rows are attractive; red rows are repulsive. Guides deposition strategy.

Why screen before DFT?

Classical dipole screening catches the strongest interactions first — in a fraction of the compute time.
